How to Find my Controllers

Here are some articles and techniques on how to find your controllers (programs) easily:

  1. Use the (Shift + F12) keyboard shortcut while running the application to pause directly into the relevant code.

By doing so you are shortening the search time for a program, and you are getting the code you need, using faster method, than tracking menus and tracking code

See using debug information in VS

  1. Use the Ctrl + T keyboard shortcut, to open visual studio's goto type dialog. (in vs 2015 it was Ctrl + ,)

  2. While running the application use the Controller List developer tools (Shift + F3) and Ctrl + T keyboard shortcut.

see: finding a controller using the developer tools and find type

  1. Right click on any class and choose Goto Definition to go to it's code. F12. If it's an interface you can choose Goto Implementation to go to the actual controller's code.

  2. In the entry point project, there is a class called ApplicationControllers that lists all controllers with their number and class info.

  3. If you get an error report from your users with the call stack, use the stack trace explorer visual studio extension to go directly from it to your code.

  4. Once you break into the code, use the Visual Studio's CallStack window to browse through the running controllers and double click any line to go to it's code. you can find the CallStack window the "Debug\Windows\CallStack" menu
